# 实验代码详细说明（中文版）

## 1. 项目总体说明

本项目是一个基于 React + Flask + MongoDB 的智能学习辅助系统，核心目标是实现从“输入学习主题”到“路线图生成、测验评估、错题重做、画像分析、统计导入导出”的完整学习闭环。

系统分为两层：

- 前端（`src/`）：负责页面交互、图表可视化、用户操作流程。
- 后端（`backend/`）：负责业务逻辑、数据持久化、鉴权与安全控制、数据分析与导入导出。

---

## 2. 代码目录与职责划分

### 2.1 前端目录（React）

- `src/pages/input/topic.js`：主题输入页，提交学习主题，触发路线图生成。
- `src/pages/roadmap/roadmap.js`：路线图展示与学习流程入口。
- `src/pages/quiz/quiz.js`：测验展示与答案提交。
- `src/pages/wrong/wrong.js`：错题管理（查询、笔记、移除、进入重做）。
- `src/pages/redo/redo.js`、`src/pages/redo/redoPlay.js`：重做列表与重做作答。
- `src/pages/profile/profile.js`：学习主页（课程概览、进度柱状图）。
- `src/pages/userprofile/userprofile.js`：学科分析页（趋势折线图、薄弱点与建议）。
- `src/pages/settings/settings.js`：账号设置、提示词管理、统计特征导入导出。
- `src/components/header/header.js`：全局导航、登出等。
- `src/utils/userManager.js`：前端登录状态与用户信息管理。

### 2.2 后端目录（Flask）

- `backend/base.py`：核心 API 入口，包含鉴权、中间流程控制、业务路由。
- `backend/mongodb.py`：数据库访问层（MongoDB 集合、CRUD、聚合逻辑）。
- `backend/database.py`：服务层封装，作为 `base.py` 与 `mongodb.py` 的桥接。
- `backend/user_profile.py`：用户画像分析器，负责统计分析与推荐生成。
- `backend/roadmap.py`、`backend/quiz.py`、`backend/generativeResources.py`：内容生成与题目相关逻辑。
- `backend/translate.py`、`backend/bilibili_search.py`：翻译与外部资源检索功能。

## 3. 后端核心机制详解

## 3.1 API 入口与请求生命周期（`backend/base.py`）

后端使用 Flask 暴露 REST 风格接口。典型请求流程如下：

1. 前端发起请求（携带 Cookie 凭证）。
2. 后端统一鉴权（解析 Bearer 或 Cookie 中 JWT）。
3. 进入具体业务路由（如测验、错题、导入导出）。
4. 调用 `database.py` 再调用 `mongodb.py` 完成数据读写。
5. 返回 JSON 响应（或 CSV 文件流）。

这一设计实现了“路由层-服务层-数据层”三层分工，便于后续维护与扩展。

## 3.2 鉴权与安全

### 3.2.1 JWT + Cookie 认证

- 登录后签发 JWT，并写入 HttpOnly Cookie。
- 受保护 API 通过 `get_user_id` 获取当前用户身份。
- 支持 Bearer 头和 Cookie 双通道读取，提高兼容性。

意义：

- 减少前端手动管理 token 的复杂度。
- 降低 token 暴露在 JS 环境中的风险。

### 3.2.2 CORS 白名单

- 仅允许配置列表中的前端源跨域访问。
- 开启 `supports_credentials=True` 以支持跨域 Cookie。

意义：

- 防止任意域名调用后端 API。

### 3.2.3 密码安全

- 注册与改密使用 `generate_password_hash`。
- 登录与敏感操作使用 `check_password_hash` 验证。

意义：

- 避免明文密码入库，提高账号安全性。

### 3.2.4 限流与安全审计

- 对高风险接口（如改密、提示词写操作、注销）做限流。
- 写入 `SECURITY_AUDIT` 审计日志（IP、UA、时间、状态）。

意义：

- 抵御暴力尝试和异常频次请求。
- 为问题排查提供证据链。

## 4. 数据库设计与实体说明

MongoDB 中至少包含以下 7 个核心实体（集合）：

1. `users`：用户基本信息、密码哈希、头像等。
2. `learning_contents`：学习内容（路线图/资源等缓存数据）。
3. `learning_stats`：学习统计信息。
4. `quiz_records`：测验记录及得分信息。
5. `wrong_questions`：错题集合与错题笔记。
6. `redo_records`：重做记录。
7. `user_profiles`：用户画像分析结果。

### 4.1 索引设计

`mongodb.py` 在初始化时创建了多类索引（唯一索引与组合索引），例如：

- 用户唯一标识索引（`user_id`、`username_lower`、`email_lower`）。
- 测验记录多字段组合索引（用户/课程/周次/子主题/时间）。
- 错题唯一索引（`user_id + question_key`）。

意义：

- 提升查询效率。
- 控制重复数据。

## 5. 核心业务流程详解

## 5.1 学习主流程

1. 用户在 `topic.js` 输入主题。
2. 后端生成路线图 `POST /api/roadmap`。
3. 用户进入测验 `POST /api/quiz`。
4. 提交答案后保存记录 `POST /api/save-quiz-record`。
5. 错题自动进入错题集（或用户手动管理）。
6. 在重做模块进行二次练习并写入 `redo_records`。

这一流程保证了“学-测-错-练-评”的闭环。

## 5.2 错题与重做管理

- 错题列表查询：`GET /api/wrong-questions`
- 错题增删：`/api/wrong-questions/toggle`、`/api/wrong-questions/delete`
- 错题笔记：`/api/wrong-questions/note`
- 重做记录增删查：`/api/redo-records`（POST/GET/DELETE）

特别说明：

- 错题使用 `question_key` 做稳定标识，避免重复插入。
- 重做记录用于追踪错题再学习效果。

## 5.3 个人资料与提示词系统

设置页集成了账号管理与提示词管理：

- 用户名、头像、密码更新。
- 账号注销并清理相关数据。
- 提示词模板新增/编辑/删除/收藏/筛选。

提示词模块是“可配置智能行为”的关键组件，支持以模板化方式复用 prompt。

## 6. 用户画像分析逻辑（`backend/user_profile.py`）

用户画像分析器 `UserProfileAnalyzer` 将学习过程数据结构化为可解释指标。

## 6.1 分析维度

1. 学习活跃度（测验频率、近期活跃、连续性等）。
2. 知识掌握度（平均分、波动、强项/弱项、趋势）。
3. 学习偏好（题型偏好、难度分布、资源偏好）。
4. 学习效果（错题率、重做率、课程薄弱点）。
5. 个性化建议（基于指标阈值生成行动建议）。

## 6.2 数据来源

- `quiz_records`
- `wrong_questions`
- `redo_records`
- `learning_contents`

## 6.3 输出结果

分析结果保存到 `user_profiles`，并可通过 `/api/user-profile`、`/api/user-profile/summary` 等接口查看。

---

## 7. 统计特征导入导出实现

## 7.1 导出

- 接口：`GET /api/user-profile/export`
- 支持格式：CSV、JSON
- CSV 结构：`section,key,value`

实现上通过 `_profile_to_csv` 将多层结构扁平化导出，便于通用数据工具处理。

## 7.2 导入

- 接口：`POST /api/user-profile/import`
- 支持格式：CSV、JSON
- 导入后写回 `user_profiles`（按当前用户覆盖画像统计）

实现上通过 `_csv_to_profile` 和 `_parse_profile_value` 做反序列化，还原数字、布尔、对象、数组等类型。

## 7.3 导入导出边界

当前导入导出对象是“用户画像统计特征”，不是全业务原始明细表的批量导入导出。

## 8. 前端可视化与交互实现

## 8.1 可视化

- `profile.js` 使用 Bar 图展示课程进度。
- `userprofile.js` 使用 Line 图展示学科趋势。

这些图表可直观表达学习结果变化，满足数据可视化要求。

## 8.2 交互特性

- 设置页支持弹窗式编辑（改密、注销、提示词编辑）。
- 导入导出在设置页统一入口，减少学习成本。

## 9. 代码工程性与可维护性说明

## 9.1 分层清晰

- `base.py` 聚焦接口与流程控制。
- `database.py` 负责服务封装。
- `mongodb.py` 负责持久化细节。

- 统一 `error_response` 结构，前后端错误消息可追踪。

## 9.3 可扩展性

- 画像分析维度可在 `user_profile.py` 继续扩展。
- 导入导出格式可扩展到更多业务模块。

## 10. 实验结果总结

本实验代码实现了一个结构完整、模块较齐全、可持续扩展的学习系统，已覆盖课程常见验收要求：

- 数据输入
- 数据管理（增删改）
- 数据查询
- 数据分析处理
- 数据导入导出（CSV/JSON）
- 数据可视化
- 统计特征导出
